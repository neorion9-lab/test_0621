# PRD

## 개요

종이 가정통신문 회신서(현장학습 신청서, 개인정보 수집 동의서 등)를 디지털로 전환하여, 수합 과정의 번거로움을 해결하고 데이터를 체계적으로 관리함.

---

## 목표 및 대상

초등학교 담임교사(운영자), 학부모(사용자).

---

## 핵심 기능

- 교사, 학부모 공통: 간편 로그인
  - 복잡한 가입 없이 인증번호나 학급 코드 기반 접속.
- 교사용: 폼생성
  - 폼 생성	텍스트, 체크박스, 서명(화면 직접 서명) 등을 포함한 폼 제작.
- 교사용: 실시간 대시보드	
  - 미제출 학생 명단 자동 분류 및 알림 발송(푸시/알림톡)
- 교사용: 데이터 추출	
  - 수합된 데이터를 엑셀/CSV 형태로 즉시 다운로드
- 학부모용: 모바일 작성	
  - 앱 설치 없이 브라우저에서 바로 제출.
- 학부모용: 상태 확인	
  - 제출 완료 시 확인 메시지 및 수정 기능(마감 전)

---

## 화면 구성

-첫화면에 교사, 학부모 로그인
-교사용 대시보드: 활성가정통신문 리스트(진행률 표시), 뻐른 생성버튼(+버튼)
-가정통신문 생성기: 제목 및 설명, 입력설명(텍스트, 날짜, 체크박스),서명란 활성화 토
-학부모 제출화면: 학생선택(드롭다운), 간편서명란, 제출버튼
-결과 분석 및 추출: 제출 목록 엑셀/csv 다운로드, 미제출 학생명단 리스트화,개별제출내용 보

---

## 기술 스택

- 프론트엔드(학부모 및 교사 화면)
  - Framework: Next.js (React)
- 백엔드 및 데이터베이스 (관리 및 수합)
  - Backend: Supabase (가장 추천)
- 서명기능
  - react-signature-canvas
- 알림(문자/알림톡
  - CoolSMS 또는 Kakao 알림톡 API

---

## 범위 (할 것 vs 안 할 것)

### In-Scope
- 반응형 웹: PC와 모바일 환경 모두에서 완벽하게 동작해야 함.

오프라인 대응: 네트워크가 불안정한 환경에서도 임시 저장이 가능하도록 구현.

기록 보존: 공문서 관리 규정에 따른 데이터 보관 기간 및 파기 기능 설정.
